# Planning Poker Voting System

## Overview
This directory contains the technical documentation for the Planning Poker voting system feature. The system enables real-time collaborative story point estimation in an agile planning context.

## Documentation Structure
- `architecture.md` - System architecture and component design
- `state-management.md` - State management and data flow
- `host-management.md` - Host role management and transfer specifications
- `voting-flow.md` - Detailed voting process and business logic
- `interfaces.md` - TypeScript interfaces and type definitions
- `socket-events.md` - Real-time event specifications

## Feature Summary
The Planning Poker voting system allows team members to participate in story point estimation through a real-time, collaborative interface. Key features include:
- Real-time voting with immediate UI updates
- Dynamic host role management
- Flexible vote modification before reveal
- Custom voting scale support
- Comprehensive error handling and edge case management
